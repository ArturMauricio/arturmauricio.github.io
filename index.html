Display: Block x Display: Inline

Display block ocupa toda a linha

display inline ocupa apenas o que é usado podendo colocar outros elementos na mesma linha
foi adicionado margin de 10px para todos os lados porem os elementos que estão definidos com display inline não respeitam heighte apenas aplicam o width
tanto faz se for margin padding ou content, apenas width é respeitado height é ignorado
para alterar um elemento basta adicionar display:inline ou display:block

Margin

Margin aceita de 1 à 4 valores, pixels ou % e até mesmo auto.
margin auto é feita automaticamente apenas para os lados.
Em display:inline as margins só vão na horizontal
Em diplay:block as margens vão para todos os lados.

ShortHand

    

    /* background properties */
    background-color: #000;
    background-image: url(images/bg.gif);
    background-repeat: no-repeat; o background sempre irá querer repetir
    background-position: left top;

    /* background shorthand*/
    background: #000 url(images/bg.gif) no-repeat left top;

    /* font properties */
    font-style: italic;
    font-weight: bold;
    font-size: .8em;
    line-height: 1.2;
    font-family: Arial, sans-serif;

    /* font shorthand */ 
    font: italic bold .8em/1.2 Arial, sans-serif;

Não irá considerar propriedades anteriores, ou seja, caso faça um shorthand, apenas ele será considerado,
quaisquer propriedades anteriores serão substituídas pelas do shorthand.
Os valores que não forem especificados irão assumir o valor padrão.
portanto caso utilize o estilo shorthand é necessário colocar todas as propriedades.
caso não coloque alguma ficará padrão mesmo que adicione logo outro por exemplo: 

		font-weight: bold;
		font: italic .8em/1.2 Arial, sans-serif;

		não reconhecerá o bold, pois ao utilizar o shorthand font todas as propriedades que
		não estejam definidas, utilizaram o padrão neste caso o padrão não é bold e sim normal
		portanto é como se escrevessemos assim:

		font:  italic normal .8em/1.2 Arial, sans-serif;

Tipos numéricos

<integer> - número inteiro como -10 ou 223
<number> - número decimal como -2.4, 64 ou 0.234
<dimension> - é um <number> com uma unidade junto: 90deg (graus), 2s (segundos), 8px (pixels)
<percentage> - representa uma fração de outro número: 50%
	
Unidades comuns

<length> - é um dos mais usados no CSS e representa um valor de distância: px (pixels), em, vw
<angle> - representa um ângulo: deg (graus), rad (radianos), turn (giro)
<time> - representa um tempo: s, ms
<resolution> - representa resoluções para dispositivos: dpi

	
Distâncias absolutas
 <length>
		São fixas e não alteram seu valor.
		
		| Unidade  | Nome                | Equivalência         |
		|----------|---------------------|----------------------|
		| cm       | Centímetros         | 1cm = 96px/2.54      | 
		| in       | Inches (polegadas)  | 1in = 2.54cm = 96px  | 
		| px       | Pixels              | 1px = 1/96th of 1in  |
		*o mais comum e mais utilizado é o px
		
		*não é recomendado usar cm
		
		
Distâncias relativas

		São relativas a um outro valor, pode ser o elemento pai, ou root, ou o tamanho da tela.
		Benefício: Maior adaptação aos diferentes tipos de tela.
		| Unidade  | Relativo a                                    |
		|----------|-----------------------------------------------|
		| em       | Tamanho da font do elemento pai               |
		| rem      | Tamanho da font do elemento raiz (root/html)  | 
		| vw       | 1% da viewport wid                            |  
		| vh       | 1% da viewport height                         |
		
		Por padrão a tag body não tem altura definida apenas largura, então quando queremos algo numa altura padrão
			utilizamos o vh por exemplo: 
			body{
				height:100vh; *irá pegar 100% da altura da viewport e aplicar em body
			}
